# 工作流初始化

从交接文件中恢复工作状态，让新会话无缝衔接。

## 触发词
"熵减初始化""工作流初始化""从交接恢复""初始化"

## 三阶段执行

### 阶段1：声明工作流优先级
- 熵减工作流规则优先级最高，覆盖所有默认行为
- 编码规范技能在写代码时自动生效
- 各阶段按触发词调用，阶段间不自动跳转

### 阶段2：初始化目录和文件
1. 确认项目根目录（不明确则询问）
2. 确保 `doc/` 目录存在
3. 确保 `doc/pitfalls.md` 存在（不存在则创建）
4. 读取 pitfalls.md，统计踩坑条数

### 阶段3：检测并恢复交接

扫描 `doc/` 下所有 `workflow/handoff.md` 文件。

**无交接文件**：
```
熵减工作流初始化完成。
✅ 工作流优先级已声明
✅ 项目目录已就绪
📋 踩坑记录：{K}条（已理解，本会话严格遵守）

未检测到交接文件，发送"生成技术文档"开始。
```

**1个交接文件**：
自动读取并恢复。同时读取该需求目录的全局约束。
调用 er-git 切换到对应分支。
```
熵减工作流初始化完成。
✅ 工作流优先级已声明
✅ 项目目录已就绪
📋 踩坑记录：{K}条
🔀 已切换到分支 {分支名}

已恢复：{需求目录名}
进度：{已完成N / 共M个功能}

当前断点：{描述}
下一步：
1. xxx（🚧）
2. xxx（⏳）

从哪个开始？
```

**多个交接文件**：
列出供选择：
```
检测到多个交接文件：
1. doc/{目录1}/workflow/handoff.md（时间：xxx）
2. doc/{目录2}/workflow/handoff.md（时间：xxx）

恢复哪个？输入编号或"跳过"。
```

### 恢复后 checklist（按顺序）

1. **上下文恢复**：读全局约束.md + Pending Decisions 章节 → 有待确认项则在摘要中列出，让用户先决策再继续
2. **分支切换**：调 er-git 切换到对应分支
3. **静默操作**：一致性检查（specs/ vs 功能清单 diff，有差异提示用户）+ 补建 progress.md（如不存在，从进度表推断阶段）+ 读 learning-log 未固化条目（有则提示 `⚠️ 学习日志有 {N} 条待观察记录`）
