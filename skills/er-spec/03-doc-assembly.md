# 阶段3：组装技术文档

> 本文件由 `SKILL.md` 调度加载，不要单独触发。
> ⚠️ **Git 操作约束**：本阶段禁止一切 git 操作。

## 前置依赖
- **必读**：`全局约束.md`
- **必读**：`specs/*.md`（所有 spec 文件）
- **必读**：需求文档（背景信息）

## 执行
1. 将所有 spec 组装为完整技术文档
2. 去重：同一张表/缓存在多个功能中出现时合并
3. 补充：架构总览、工期评估、上线 CheckList
4. 一致性检查：全文格式、术语与全局约束一致
5. 保存为 `技术文档.md`

## 技术文档模板

```markdown
# [需求名称] 技术文档

> 本文档用于团队评审

## 1. 背景与目标
### 背景
### 目标
### 非目标（边界）

## 2. 方案设计
### 整体架构
### 核心流程
### 关键技术点

## 3. 数据设计
### 表结构
### 缓存设计（如有）

## 4. 接口设计

## 5. 工期评估
| 任务 | 工期 | 说明 | 负责人 |

## 上线 Check List
### 发布灰度流程
### 上线流程
```

## 工期评估要求
- 负责人列保留，值留空
- 基于实际开发模式评估

## 上线 CheckList 要求
- 面向部署流程，不是功能测试清单
- 分"灰度"和"上线"两部分
- 每部分只列主要步骤

## 完成后
**效能打点**：为所有功能追加 `- [{时间}] spec | 技术文档组装完成，spec 阶段结束`。
静默更新 `workflow/progress.md`：当前阶段改为 `er-dev`，当前功能为依赖顺序第一个。
