# AI 代码质量红线

> 从实际错误中沉淀的硬性约束。每条都是 er-dev 阶段的强制规则，违反即为缺陷。
> 此清单通过学习日志机制持续补充。

- **禁止循环操作外部存储**：DB/缓存/RPC 的读写都不允许在 for 循环中逐条执行，必须用批量接口（Pipeline、MGET、批量 INSERT 等）。如果底层没有批量接口，先补充
- **禁止伪造不确定的值**：URL、域名、密钥、配置项等，如果不确定真实值，先看现有代码中同类场景怎么处理。找不到就标 TODO，绝不编造
- **禁止跳过已有抽象层**：先看已有代码中同类场景的调用链路（如数据获取走哪一层），新功能保持一致，不要绕过已有的封装直接调底层
- **禁止重复实现已有功能**：前置步骤中已列出的可复用代码，实现时必须优先调用，不允许重新实现功能相同的方法。如果已有方法不完全满足需求，应扩展已有方法而非新建
- **禁止在 er-dev 阶段执行任何 git 命令**：包括 git status / git add / git commit / git push / git stash。所有 git 操作由 er-test 阶段统一执行。编译通过、代码写完都不是 commit 的理由
