# 学习固化工具

> 各阶段 skill 检测到信号后调用本工具，传入信号内容。本工具负责记录格式、追加逻辑、固化提议。

## 调用方式

调用方传入：
- `skill_name`：哪个 skill 产生的信号（如 er-dev、er-spec）
- `signal`：信号描述（用户改了什么 / 发现了什么问题）
- `cause`：原因分析
- `suggestion`：改进建议

## 记录

在 `~/.claude/skills/{skill_name}/learning-log.md` 追加：

```markdown
### YYYY-MM-DD | 需求名/功能名
- **信号**：{signal}
- **原因**：{cause}
- **建议**：{suggestion}
- **状态**：⏳
```

## 固化提议检测

追加记录后，检查该 learning-log.md 中是否存在**同类问题在不同需求或不同功能模块中出现 2 次以上**（⏳ 状态）。

判断"同类"的标准：
- 同一个错误模式，发生在**不同的需求目录**下，或
- 同一个错误模式，发生在**不同功能模块**（非同一功能的两次迭代）

**不满足跨需求/跨模块条件时，不触发固化提议。**

满足条件时：
- 在任务结束时附上提议：说明观察到的模式、建议的 skill 修改内容
- **必须等待用户明确回复"确认"或"同意"后，才能修改 SKILL.md**
- 用户未回应或回应模糊则不修改，不再追问

## 日志维护

未固化条目（⏳）超过 20 条时，提醒用户清理（否决标记 ❌ 或确认固化 ✅）。
